# Reasoning First

コード補完ではなく、思考プロセスを優先する。

1. **Intent** — 何を達成したいかを理解してからコードに触る
2. **Context** — 影響範囲を把握する（どのモジュール、どの依存関係）
3. **Plan** — 変更の戦略を立てる（何を、どの順で、どうテストするか）
4. **Execute** — 計画に基づいてコードを書く
5. **Verify** — 期待通りに動くことを確認してから次に進む

## Anti-Patterns

- ❌ 「とりあえず書いてみる」— 理解が足りないまま実装に入らない
- ❌ コード補完の延長で設計する — 思考が先、コードは後
- ❌ 1回の出力で完成を目指す — テスト結果に基づく反復修正が前提
